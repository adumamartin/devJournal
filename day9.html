April 22

I've decided to participate in the #100DaysOfCode trend on Twitter as a result of the quarantine imposed during the COVID-19 pandemic. I think I will be using this page as a reference point to when I started and to measure the development of my ability.

I have been using lessons from freeCodeCamp and as of today have completed the 'Basic HTML' and 'Basic CSS' portions. From hereon forward, I will be updating future iterations of this page with new skills I learn to implement in addition to side (main?) projects to practice said skills.

Thinking ahead, for ease of future access and navigation I plan to either implement a task bar or banner along the top or one side of the site.

A top banner offers several benefits:

  • Retains a sense of uniformity across different mediums (web vs mobile) as I feel side banners take up valuable horizontal space on mobile and I usually find them cumbersome.
  • Allows a quick cursory glance at the page and contains everthing a visitor would need to quickly navigate elsewhere.

As I currently envision this project to be a progress journal of sorts, I feel a side banner as a type of table of contents can be where other iterations of the page are linked. I could even include links to other projects all in one general area for better QoL for myself/visitors.

Today I've decided to create a github repository and begin uploading each day's work onto github. I did not originally think of uploading into github so I had to copy my original files into the location that my repository was saved in locally.

I used solely terminal command line to clone my repository and add/commit files, but I relied on the trusty copy/paste for moving the original files as was not yet sure of my ability to move files across directories using command line.

It's a bit annoying that I have to scroll down everytime I want to see my updates on the browser so I'm strongly considering shifting the format of this journal to reverse chronological order.

Alternatively, I could also forego this current scrolling journal format and use on in which the entire page is dedicated to that day's work.

  • I personally prefer the current iteration because when I eventually have enough entries, it'll be aesthetically pleasing to scroll and see the changes rather than clicking through multiple webpages.
  • Instead of literally coding in reverse order, I'm hoping I will learn something that will allow the process to be automated and the entries displayed reverse chronologically behind the scenes between text editor and browser.
    • Perhaps something as simple as taking each entry as an element in an array list and then rendered in reverse order when displayed. I am not sure if this is a thing in web development.
